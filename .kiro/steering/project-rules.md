# 项目规则 / Quy Tắc Dự Án

## 核心原则 / Nguyên Tắc Cốt Lõi

### 1. 深思熟虑，认真执行 / Suy Nghĩ Kỹ Càng, Thực Hiện Nghiêm Túc
- **必须仔细思考**用户的每个要求
- **不允许敷衍了事**或草率完成工作
- **必须完整实现**所有功能，不能遗漏任何部分
- **必须测试验证**代码能够正常运行

### 2. 完整性要求 / Yêu Cầu Tính Hoàn Chỉnh
- 创建文件时必须包含**完整的代码**，不能只写部分
- 所有功能必须**完全实现**，不能留空或使用占位符
- 所有导航和按钮必须**真实可用**，能够正确跳转
- 所有依赖项必须**正确配置**在 package.json 中

### 3. 代码质量标准 / Tiêu Chuẩn Chất Lượng Code
- 代码必须**语法正确**，没有错误
- 必须遵循**最佳实践**和编码规范
- 必须添加**适当的注释**说明复杂逻辑
- 必须确保**类型安全**（TypeScript）

### 4. 用户体验优先 / Ưu Tiên Trải Nghiệm Người Dùng
- 所有交互必须**流畅自然**
- 界面必须**美观一致**
- 响应必须**快速准确**
- 错误处理必须**友好清晰**

## 工作流程 / Quy Trình Làm Việc

### 步骤 1: 理解需求 / Hiểu Yêu Cầu
- 仔细阅读用户的要求
- 分析需要实现的功能
- 确认技术栈和依赖项
- 规划文件结构

### 步骤 2: 完整实现 / Thực Hiện Hoàn Chỉnh
- 创建所有必需的文件
- 编写完整的代码（不留空白）
- 实现所有功能和交互
- 确保所有导航链接正常工作

### 步骤 3: 验证测试 / Kiểm Tra Xác Thực
- 检查语法错误
- 验证功能完整性
- 测试所有交互流程
- 确认依赖项正确安装

### 步骤 4: 文档说明 / Tài Liệu Hướng Dẫn
- 提供清晰的使用说明
- 说明如何运行项目
- 列出所有功能特性
- 记录可能的问题和解决方案

❌ **绝对不允许：**
- 敷衍了事，草率完成
